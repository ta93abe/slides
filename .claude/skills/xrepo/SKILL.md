# xrepo — 兄弟リポジトリ参照・操作スキル

このリポジトリのナレッジコンテキストを保持したまま、外部リポジトリを探索・操作する。

## コンセプト

- **ナレッジが一番濃いリポジトリをホームにして作業する**
- モノレポにできない事情があっても、まるでモノレポであるかのように扱える
- 読み取りだけでなく書き込みにも対応
- コーディング、戦略文書、デザイン文書、マーケティングコピーなど、すべてがAIの視界に入る

## 引数

| パラメータ | 必須 | 例 | 説明 |
|---|---|---|---|
| パス | Yes | `../other-repo` | 対象リポジトリへの相対パスまたは絶対パス |
| 指示 | No | `READMEを日本語に翻訳して` | 対象リポに対して行いたい作業。省略時は探索モード |

## 手順

### 1. 対象リポジトリの探索

まず `rg --files` でファイル構造を把握する。

```bash
rg --files <path> --sort path
```

大きなリポジトリの場合は以下で絞り込む:

```bash
# 主要ファイルのみ
rg --files <path> --sort path -g '!node_modules' -g '!.git' -g '!dist' -g '!vendor' -g '!__pycache__' | head -100

# 特定の拡張子
rg --files <path> -g '*.ts' -g '*.md'
```

### 2. コンテキストの構築

対象リポの以下を読んで全体像を掴む（存在する場合）:

- `README.md` / `CLAUDE.md`
- `package.json` / `pyproject.toml` / `Cargo.toml` 等
- ディレクトリ構造（上記の `rg --files` 結果）

### 3. ナレッジの活用

このリポジトリ（ホーム）が持つナレッジを最大限に活かす:

- **CLAUDE.md** — プロジェクトの規約・アーキテクチャ方針
- **memory/** — ユーザーの好み・フィードバック・プロジェクト文脈
- **~/zettelkasten** — Obsidian Vault のノート群

指示に関連するキーワードで Glob/Grep を使い、ホーム側の知識を参照してから対象リポの作業に取り掛かる。

### 4. 作業の実行

指示がある場合はそれに従い、対象リポのファイルを読み書きする。

- **読み取り**: Read / Grep / Glob で対象リポのファイルを直接参照
- **書き込み**: Edit / Write で対象リポのファイルを直接編集
- パスは絶対パスに解決してからツールに渡す

### 5. 差分の確認

書き込みを行った場合、対象リポで差分を確認する:

```bash
git -C <path> diff
git -C <path> status
```

## 注意事項

- 対象リポの `.gitignore` や既存の規約を尊重する
- 対象リポへのコミット・プッシュはユーザーの明示的な指示がない限り行わない
- 大きな変更を行う前に方針をユーザーに確認する
