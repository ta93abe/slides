# R2 で何が変わるか

**Before**: データの置き場がクエリエンジンを決める

```
S3 → Athena（AWSに縛られる）
GCS → BigQuery（GCPに縛られる）
```

**After**: データの置き場とクエリエンジンが分離する

```
R2 → DuckDB（手元で）
R2 → Snowflake（ガバナンスが必要なら）
R2 → R2 SQL（軽い集計なら）
R2 → Spark（重い処理なら）
```

> **エグレス $0 = クエリエンジンの自由選択権**

# R2 の正直な評価

### 勝ってるところ

- エグレス $0（圧倒的）
- ストレージ単価 35% 安い
- Iceberg REST Catalog が標準装備
- S3 互換 API（既存ツールがそのまま使える）

### 負けてるところ

| 機能 | S3 | R2 |
|---|---|---|
| バージョニング | ✅ | **❌** |
| Glacier 相当 | ✅ | ❌（InfrequentAccess のみ） |
| Access Points | ✅ | ❌ |
| レプリケーション制御 | ✅ | ❌ |

> **バージョニングがない** = 誤上書きしたら終わり。これがエンタープライズ採用の最大の壁
